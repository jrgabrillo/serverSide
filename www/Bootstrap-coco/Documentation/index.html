Overview

Coco template is created to work smoothly on all devices. It is a backend interface for your application which you can use on all your devices and even create application that your users can use your application via iPhone, iPad, Android or any other devices. It is not just a template that you can use with your software but also have applications that you can use inside your application like live weather widget which gets weather information from Yahoo according to your location, Todo list that you can actually use on your daily activities or calculator that you can use when you are doing your accounting.

It has some game changer features that will make your application much useful than ever. Now lets take a look at the features list.

Very well organized code

Template is coded with a very clean and understandable coding even for non developers.

If you study Twitter bootstrap documentation and get used to it's structure, it will be a baby toy for you to set up your application using this template.

Portable & actually usable widgets

Coco template has widgets that actually works as you need!

In template, you can make your widgets portable. Not just from one grid to another. You can actually carry widgets anywhere you want in the template or resize it to the size you need. We thought it would be cool if you can for example move calculator to up, next to daily orders table and make calculations without the need to use any other application. This will work on your tablets as well which is normally a problem. When you are viewing a page, you can open a calculator aside and do your calculation when you are using your PC or MAC. However most of the tablets or phones are not able to run a second application when you are browsing a page.

Don't wait any more and try it on your own device like it is on the screenshot aside. Make it portable and move it up. All operations are touch friendly!

	function connect_db(){
		global $config;
		$db = mysql_connect($config['database']['db_host'].":".$config['database']['db_port'],$config['database']['db_user'],$config['database']['db_pass']) or die("Unable to connect mysql server!");
		$conn = mysql_select_db($config['database']['db_name']) or die("Database not found in mysql server!");
		return $conn;

	function menu_from_query($query){
		global $index;

		$result	=	mysql_query($query);

		$menu = array(
			'items' => array(),
			'parents' => array()
		);

		while ($items = mysql_fetch_assoc($result)){
			$menu['items'][$items['id']] = $items;
			$menu['parents'][$items['parent']][] = $items['id'];
			$parent = $items['parent'];
			while($parent != 0){
				if($menu['items'][$parent]['parent'] == 0)
					break;
				$parent = $menu['items'][$parent]['parent'];
			}
			$menu['items'][$items['id']]['top_parent'] = $parent;
			if($index == $items['link'] && $parent != 0)
				$menu['items'][$parent]['class'] = "open";
		}
		return menu_builder($menu);
	}

	function menu_builder($menu, $parent = 0){
		global $index;
		$output = "";
		if (isset($menu['parents'][$parent])){
			$output .= "
    ";
			foreach ($menu['parents'][$parent] as $itemId){
				$active = ($index.".html" == $menu['items'][$itemId]['link'])?" class='active'":null;
				if(!isset($menu['parents'][$itemId])){
					$output .= "
  • ".(!empty($menu['items'][$itemId]['icon'])?"":null)."".$menu['items'][$itemId]['title']."
    • ";
				}
				if(isset($menu['parents'][$itemId])){
					$dropdown = ' ';
					$output .= "
  • ".(!empty($menu['items'][$itemId]['icon'])?"":null)."".$menu['items'][$itemId]['title']."$dropdown";
					$output .= menu_builder($menu, $itemId);
					$output .= "
    • ";
				}
			}
			$output .= "
";
		}

		return $output;
	}
?>

After including the code in your script, make sure your page has $index variable setted up in your page and you can display your menu with calling:

$menu = menu_from_query("SELECT id, title, link, icon, parent FROM menu ORDER BY parent, sort, title");

Usage Tips

Simple tips for using template

  1. To make left sidebar small by default add "enlarged forced" to #wrapper element
  2. To make left sidebar fixed by default add "fixed-left" to body element
  3. To make right sidebar open by default add "open-right-sidebar" to #wrapper element
  4. To have toolbar on left side of the widget header, you can add a div with "additional-btn left-toolbar" class
  5. To make an interface contain movable widgets, you should add "portlets" class to the container item
  6. Left sidebar also have spots for widgets. You can write your own or you can simply drag & drop sizable widgets like calendar to left sidebar
  7. To use a widget as an application, you need to:
    1. Make sure your widget has popout button.
    2. Make sure your widget is working and all needed libraries are loaded to the page.
    3. Make sure you add the application link on app toolbar using the same structure with the following example:
      <a data-status="active" data-app="todo-app" href="javascript:;"><i class="icon-check"></i>Todo List</a>
    4. data-app should have the same value with the ID of your widget. data-status is active by default which means the widget is visible. To make it hidden by default, make the value data-status="inactive"
  8. There are 6 widgets in system that can act as "app" for now. They are Notes, Todo list, Weather, Calendar, Calculator and Realtime Stock Chart. You can make them visible or invisible by default
